Add Repo:
=========

1. When selecting the "Add Repo" option, you will be asked for the repo URL and do the validation like today accordingly.
The URL validation:
1.1. It needs to be that the url exists, (not empty) - if empty - Throw an error and let the user recover.
1.2. If the URL is not related to GitHub and not in the repo pattern of a GitHub repository - Throw an error and let the user recover.
1.3. If the Repo not exists - Throw an error and let the user recover ("Please enter the repo's URL once its created").
1.4. Validate that the repo name is extracted properly.
1.5. If the repo exists - Continue to the next step.

2. The next step - Descriptions
2.1. The CLI will ask the user to enter a description for the README.md file.
This is a require field and minimum of 340 and maximum of 600 characters - Let the user recover in case of any error.
2.2. The CLI will ask the user for the package.json description.
This is a require field and minimum of 120 and maximum of 300 characters - Let the user recover in case of any error.
2.3. The CLI will ask the user for the GitHub project description.
This is a require field and minimum of 340 and maximum of 350 characters - Let the user recover in case of any error.
2.3. Once all the descriptions filled - Continue to the next step.

3. The next step - Keywords / Topics
3.1. The CLI will ask the user for keywords / topics.
This is a require field and minimum of 8, maximum 20 - Let the user recover in case of any error.
Remove duplicates and validate the minimum and maximum after duplicates were cleared.

3. The next thing to do is to clone the repo and create the folder, like today - The logic already exists.
Don't need to change anything.

4. The next step is to inject all the templates:
4.1. .gitignore - No changes needed.
4.2. CHANGELOG.md - No changes needed.
4.3. CODE_OF_CONDUCT.md - No changes needed.
4.4. CONTRIBUTING.md - Changes needed:
4.4.1 Replace the "#REPO-NAME#" with the repo name from the given URL - Logic exists today, make sure not to break it.
4.5. LICENSE - Changes needed:
4.5.1 Replace the "#YEAR#" with the current year - Logic exists today, make sure not to break it.
4.6. SECURITY.md - No changes needed.

5. The next step is to inject the package.json with the following changes:
5.1. Replace:   "name": "#REPO-NAME#", with the repo name extracted from the URL.
5.2. Replace:     "url": "git://github.com/orassayag/#REPO-NAME#.git", with the repo name extracted from the URL.
5.3. Add all the keywords / topics in "  "keywords": [],"
5.4. Get the latest versions of the listed packages and fill them in the dependencies and devDependencies sections.
5.5. Run "pnpm i" and verify the creation of "pnpm-lock.yaml" abd "pnpm-workspace.yaml" files.

6. Commit and push to the newly repo:
6.1. Once done, "git add ." all the new files.
6.2. After that, "commit -m 'Initial release with core features'".
6.3. After that, "git push --force-with-lease".

7. At this point console log "Repo created successfully".

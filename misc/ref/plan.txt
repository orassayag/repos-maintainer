repos-maintainer:
=================

Goal:
The goal of the project is to maintain the existing repos and a tool to add and update existing repos to keep them with hight structure standard

This project will have CLI steps and 1 auto script.

Once we enter "pnpm run start" we will perform the validation steps.

-Validation steps:
-Validate that the C:\Or\web\projects exists - declared on settings.ts file - if not exists / empty - throw an error.
-Valudate that the C:\Or\web\project-repos-names.txt - declared on settings.ts file - if not exists / empty - throw an error.
-Validate that connected to GitHub account. If not - throw an error.
-Once all valid - we can continue.

Manual script (CLI steps):
-We will use the menu selection like in the event & people sync project.
-So once the validation passed, the user will see the following:

===Repos Maintainer===

? Select a script to run (ESC to exit): ...
> 🔄 Add Repo       	 - Add the initial features missing for a GitHub project
  ♻️ Repos Sync          - Syncs all the repos
  🚪 Exit

Add Repo:
=========
1. When selecting this option, the CLI will ask the usr for the newley repo URL. If the url is invalid (not github url, missing repo name in the url) - An error will be throwen but the use will be able to recover and give the correct url.
2. When the URL is valid - We will do a simple API call to GitHub API to check if the repo exists within the given name. If not - throw an error.
3. If the repo exists we should do the following:
-Check if the project is listed in the C:\Or\web\project-repos-names.txt file - If not, add it according to the existing patterns there and add it in the alfabetifc correct order.
-Also check that the url and the project name are correct in C:\Or\web\project-repos-names.txt file according to the data fetched from GitHub. If not - Fix them accordinglly.

For the following steps, we will do them all and not stop if any or all of the steps fail:
